Multi-Agent Inference Reliability Framework

What happens when the model starts failing at runtime and there is no human in the loop to catch it? That's my research problem. I drove it as first author alongside 2 faculty co-authors, from question to architecture to statistics. The answer was a 3 agent monitor that catches and corrects bad LLM outputs without retraining.

The architecture

3 agents and a Reversion gate

A Planner writes the code. A Critic scores it for hallucination risk between 0 and 1. If that score crosses a threshold the Fixer rewrites it and a Post-Critic scores the result. Then a reversion gate checks whether the rewrite actually improved things and throws it away if it did not. Everything runs locally through Ollama. No API calls, no fine-tuning, fully reproducible.

Fresh Planner output, not yet scored.
Cleared. Either it scored below τ = 0.70 and skipped the Fixer, or its rewrite survived the gate.
Flagged at or above τ. Sent to the Fixer for a rewrite.
Rewrite reverted. The gate threw it away and kept the original, which is what happens to 73.4% of them.

A baseline of Planner straight to Post-Critic runs alongside every condition, so every result is a clean before and after. The operating threshold τ settles at 0.70.

Result

The deltas.

Across 10 phases and 5 model families, the story was never a single average gain. The number I care about is the failure condition: a weak model (Code Llama 7B) gained +30.3 points pass@1 while a strong one (Qwen2.5 Coder 7B) lost 4.0, both statistically significant. I call it the Inverse Capability Hypothesis. Below roughly 65 percent baseline monitoring is worth deploying; above that the benefit fades, and the exact crossover is still an open question. Knowing when the monitor starts doing harm matters as much as the monitor itself.

Making it defensible

A single run has no error bars, so reviewers would not buy it

A deterministic run has zero variance and a number with no confidence interval will not survive peer review at a serious venue. So I reran the 2 endpoints as 3 genuinely independent trials each with a non-deterministic planner. Both intervals exclude zero. The gain and the regression are both statistically defensible, not a pair of lucky runs.

What actually does the work

I turned each piece off to see which one carried the result

Between the net-negative early phases and the net-positive turn, 3 things changed at once. An 8B fixer, the problem statement handed to the fixer and the reversion gate. So I turned each one off in isolation on Code Llama to see which one actually mattered.

The gate surprised me. On a weak model it slightly lowers raw pass@1, because almost any fix helps when you start at 20 percent. Its real value shows up on strong models, where it reverts bad rewrites and keeps the monitor from doing harm. Handing the fixer the problem statement was the single biggest lever on the weak model.

Status

Where it stands

All 10 experiment phases are done. I am writing it up as first author with Prof. Vahid Alizadeh and Prof. Noriko Tomuro as faculty co-authors at DePaul, currently under peer review at a top-tier venue. The Phase 10 cross-benchmark runs were built to test whether the claim holds beyond HumanEval and it did. The law held on both HumanEval+ and MBPP.
