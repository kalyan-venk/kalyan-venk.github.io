Multi-Agent Inference Reliability · Blueprint

System design & Code map Visuals

2 live blueprints of Multi-Agent Inference Reliability, my first-author research on whether a runtime monitor makes an AI coding agent better. The first traces the stages of progression: how a Planner, Critic and Fixer turn a problem into checked code. The second is the code map: every file in the repo, and how one run becomes the next result on disk.

Hover anywhere on a diagram and a lens magnifies it, so you can read every label without shrinking the picture. The moving dots are data in flight, coloured by what kind of data it is.

The finding is the failure condition

Monitoring is not a free win. Below roughly 65% baseline it lifts a weak agent a lot; above it, the same layer quietly costs a strong one points. Knowing which side your model sits on is the contribution.

The gate is do-no-harm, not the hero

The reversion gate throws away 73.4% of the Fixer's rewrites, keeping only strict improvements. The ablation shows problem context and fixer scale drive the gains; the gate mostly protects the downside.

The critic is a hybrid

A static AST pass catches syntax errors and undefined names first and sets a floor. The LLM critic can only push the risk score higher, never talk a real defect down to clean.
